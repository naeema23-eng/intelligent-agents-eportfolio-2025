What I did Evidence + learning

Activity 6: Creating Agent Dialogues (KQML + KIF)

Focus: constructing a structured agent dialogue using KQML performatives and KIF-style content to support autonomous communication between agents.

What I did Dialogue + learning

Approach

  • Created a dialogue between Alice (procurement agent) and Bob (warehouse stock control agent).
  • Used KQML performatives (ask-one, tell) to represent intent.
  • Represented message content using KIF-style logical expressions and variables (?qty, ?ports).
  • Specified a shared ontology (Warehouse-Stock) to reduce semantic ambiguity.

Agent Dialogue (KQML + KIF)

(ask-one
  :sender Alice
  :receiver Bob
  :language KIF
  :ontology Warehouse-Stock
  :content (available-stock (item "Television") (size-inches 50) ?qty))

(tell
  :sender Bob
  :receiver Alice
  :language KIF
  :ontology Warehouse-Stock
  :content (= ?qty 18))

(ask-one
  :sender Alice
  :receiver Bob
  :language KIF
  :ontology Warehouse-Stock
  :content (hdmi-slots (item "Television") (size-inches 50) ?ports))

(tell
  :sender Bob
  :receiver Alice
  :language KIF
  :ontology Warehouse-Stock
  :content (= ?ports 3))
Evidence
Completed KQML/KIF dialogue showing knowledge-level interaction and ontology alignment.
Outcome
Practical understanding of ACL communication and why semantic design (ontology) is essential for interoperability.

Activity 8: Deep Learning in Action (Unit 10)

Focus: researching a deep learning application likely to impact society and evaluating ethical, privacy, and socio-technical implications.

What I did Overview + impacts

Chosen application

Deep learning for medical imaging diagnosis (e.g., detecting breast cancer and other conditions from radiology images).

Overview (what it does)

Deep learning models can support clinicians by analysing medical images (such as mammograms, CT, or MRI) to identify patterns associated with disease. These systems aim to improve sensitivity for early detection, reduce missed findings, and prioritise cases for review, especially in high-volume clinical settings.

How it works (brief synopsis)

Most diagnostic imaging systems use convolutional neural networks (CNNs) or related architectures trained on labelled image datasets. During training, the model learns features that correlate with outcomes (e.g., malignant vs benign). At deployment, the model produces probability scores and may provide localisation cues (e.g., heatmaps) to support interpretation and auditing.

Potential socio-technical impacts

  • Ethics & fairness: performance can vary across demographic groups if datasets are unbalanced, potentially amplifying health inequalities.
  • Privacy & security: sensitive health data requires strict governance; model training and storage must respect consent and data minimisation.
  • Accountability: errors raise questions about responsibility (developer, hospital, clinician); governance must define oversight and escalation routes.
  • Social good: when validated properly, systems may support earlier detection and more consistent screening, improving outcomes.
Evidence
“Deep Learning in Action” forum post (Unit 10), referenced and linked in the Discussions section.
Outcome
Improved ability to evaluate deep learning deployments using both technical criteria and socio-technical risk framing.

References

  • Esteva, A. et al. (2017) ‘Dermatologist-level classification of skin cancer with deep neural networks’, Nature, 542, pp. 115–118.
  • Topol, E. (2019) Deep Medicine: How Artificial Intelligence Can Make Healthcare Human Again. New York: Basic Books.
  • Obermeyer, Z. and Emanuel, E.J. (2016) ‘Predicting the future — big data, machine learning, and clinical medicine’, New England Journal of Medicine, 375(13), pp. 1216–1219.

Activity 9: Creating Parse Trees (Unit 8)

Focus: representing language structure using parse trees and connecting syntax to meaning in NLP tasks.

What I did Artefact + learning

I completed the “Creating Parse Trees” activity by constructing parse trees for sentences and identifying how grammatical structure (e.g., noun phrase, verb phrase, prepositional phrase) supports interpretation in NLP pipelines. This reinforced how language can be represented formally before higher-level reasoning or learning components are applied.

Example artefact (illustrative parse)

Sentence: “The agent analysed the evidence in the archive.”

(S
  (NP (DT The) (NN agent))
  (VP (VBD analysed)
      (NP (DT the) (NN evidence))
      (PP (IN in)
          (NP (DT the) (NN archive))))
  (. .))
Evidence
Completed parse trees (Unit 8 activity) and brief written explanation (included as e-portfolio evidence).
Outcome
Improved understanding of syntactic structure and how it supports NLP interpretation and downstream intelligent reasoning.
